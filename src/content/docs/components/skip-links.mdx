## Introduction

The Skip Links component provides a way for keyboard and screen reader users to bypass repetitive navigation elements and jump directly to the main content. It's a crucial accessibility feature that becomes visible only when focused.

<Aside type="caution" title="Required markup">
  Your page must have either an element with `id="main-content"` or an `<h1>` element for the skip link to work properly.
</Aside>

## When to use

Skip Links are essential for keyboard navigation in these scenarios:

- At the top of every page
- Before navigation menus
- On pages with repetitive content
- Content-heavy websites
- Complex navigation structures
- Any site that prioritizes keyboard accessibility

## Usage

Learn how to implement the Skip Links component in your project:

```astro
---
import { SkipLinks } from 'accessible-astro-components'
---

<SkipLinks />

<!-- Target element -->
<main id="main-content">
  <h1>Page title</h1>
  <!-- Your content -->
</main>
```

<Aside type="tip" title="Best practice">
  Place the Skip Links component as the first focusable element on your page, before any navigation or content.
</Aside>

## Props

<div class="props-wrapper">
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `text` | `string` | `'Skip to main content'` | Custom text for the skip link |
| `target` | `string` | `'#main-content'` | Target element ID to skip to |
| `class` | `string` | `''` | Additional CSS classes to apply |
</div>

## Accessibility

The Skip Links component prioritizes accessibility through several features:

- Appears only when focused (visually hidden otherwise)
- Provides keyboard-only navigation
- Uses high contrast colors for visibility
- Clear focus indicators
- Semantic HTML structure
- Proper focus management
- Fallback to `<h1>` if main content ID is missing
- Console warning if neither target is found
