## Ways to contribute

There are many ways to contribute to Accessible Astro:

1. **Code contributions**
   - Fix bugs
   - Add new features
   - Improve documentation
   - Write tests

2. **Documentation**
   - Fix typos
   - Add examples
   - Clarify explanations
   - Translate content

3. **Issue reporting**
   - Report bugs
   - Suggest features
   - Share use cases
   - Ask questions

4. **Community support**
   - Answer questions
   - Share knowledge
   - Review pull requests
   - Help others

## Getting started

1. Fork the repository you want to contribute to:
   - [Accessible Astro Components](https://github.com/markteekman/accessible-astro-components)
   - [Accessible Astro Starter](https://github.com/markteekman/accessible-astro-starter)
   - [Accessible Astro Dashboard](https://github.com/markteekman/accessible-astro-dashboard)

2. Clone your fork:
```bash
git clone https://github.com/YOUR-USERNAME/REPO-NAME.git
```

3. Create a new branch:
```bash
git checkout -b fix/issue-description
```

<Aside type="tip" title="Branch naming">
  Use prefixes like `fix/`, `feature/`, or `docs/` to categorize your changes.
</Aside>

## Making changes

1. Follow our [development guidelines](/contributing/development)
2. Make your changes
3. Add or update tests if needed
4. Update documentation if needed
5. Follow our [style guide](/contributing/style-guide)

## Submitting changes

1. Commit your changes:
```bash
git add .
git commit -m "fix: description of your changes"
```

2. Push to your fork:
```bash
git push origin fix/issue-description
```

3. Open a pull request:
   - Use a clear title
   - Reference any related issues
   - Describe your changes
   - Add screenshots if relevant

<Aside type="note" title="Pull requests">
  All pull requests should target the `main` branch unless specified otherwise.
</Aside>

## Code review process

1. Maintainers will review your PR
2. Automated tests will run
3. Changes might be requested
4. Once approved, your PR will be merged

## Need help?

- Check our [development guide](/contributing/development)
- Read about [reporting issues](/contributing/issues)
- Join our [GitHub discussions](https://github.com/markteekman/accessible-astro-components/discussions)
- Review our [style guide](/contributing/style-guide)

<Aside type="tip" title="First time?">
  Look for issues labeled `good first issue` to get started with contributing.
</Aside>
